# Quick Design Spec: HUD

**Type**: New Small System
**Scope**: DOM-based overlay that displays live run stats and gameplay state. HP bar, distance, crystal count, super-suit cooldown ring, current planet label, +score popup animations on pickup/obstacle break, and a subtle hit-flash on damage. Polls Run Lifecycle and Super-Suit Combat for state; subscribes to pickup/hit events for feedback. Contains zero gameplay logic.
**Date**: 2026-04-09
**Estimated Implementation**: ~3 hours (M effort)

---

## Overview

HUD is a DOM overlay rendered on top of the canvas. No Three.js, no Canvas2D — just styled `<div>` elements with CSS animations. This keeps it simple, fast (DOM is lighter than a full Three.js render layer for 5-10 text elements), and makes iteration trivial.

The HUD updates via `requestAnimationFrame` (not the engine's fixed tick) because UI smoothness wants real frame rate. Each frame: read stats from Run Lifecycle, update DOM text/widths, tick any active popup animations.

**Note (2026-04-09 simplification):** Space Runner uses an on-touch death
model. There is no HP bar in the HUD — touching any obstacle ends the run
immediately. The HUD shows distance, crystals, score, super-suit cooldown,
and planet label only. Hit flash is also removed (death itself is the
feedback).

## Core Rules

### 1. DOM structure

```html
<div id="hud" class="hud">
  <div class="hud-top-left">
    <div class="hud-distance">0 m</div>
  </div>

  <div class="hud-top-right">
    <div class="hud-planet-label">Rocky · Planet 1</div>
    <div class="hud-crystals"><span class="icon">◈</span> 0</div>
    <div class="hud-score">0</div>
  </div>

  <div class="hud-bottom-right">
    <div class="hud-supersuit">
      <svg class="cooldown-ring">...</svg>
      <span class="label">SUPER</span>
    </div>
  </div>

  <div class="hud-popups"></div>
</div>
```

No HP bar, no hit-flash overlay (on-touch death — see Overview).

Injected into `<body>` on creation. Removed on dispose.

### 2. Updates per frame

In `requestAnimationFrame`:
1. `stats = runLifecycle.getStats()`
2. Update distance text: `distanceEl.textContent = stats.distance + ' m'`
3. Update crystals text: `crystalsEl.textContent = stats.crystalsCollected`
4. Update score text: `scoreEl.textContent = stats.score`
5. Read `superSuit.getCooldownFraction()` → update SVG cooldown ring
6. Tick any active popup animations (advance life, apply transform)

Text updates are cheap because DOM text diffing is internal to the browser. We don't need memoization for jam scope.

No HP read — on-touch death removed the bar. Player death is handled by the
Results Screen transition, not the HUD.

### 3. Popup animations

On `pickups.onPickup('crystal', 10)` or `superSuit.onObstacleBroken(type)`:
1. Create a new DOM `<div class="hud-popup">+10</div>` or `+25`
2. Position at a random spot near the crystal / obstacle (or just near the HUD score element)
3. CSS animation fades up + out over 1 second
4. Remove from DOM on animation end

### 4. ~~Hit flash~~ — REMOVED

Removed in the on-touch death simplification. Any obstacle touch is fatal,
so the Results Screen transition IS the "you got hit" feedback. No need
for an intermediate flash.

### 5. Planet label updates

On `planetCheckpoint.onPlanetChanged(biome, planetIndex)`:
1. Update `.hud-planet-label` text: `${capitalize(biome)} · Planet ${planetIndex + 1}`
2. Play a brief slide-in CSS animation on the label (fresh planet feel)

### 6. State-based visibility

HUD is visible ONLY when Run Lifecycle state is `running`. In `title`, `dead`, or `results`, HUD is hidden via `display: none`.

Subscribes to `runLifecycle.onStateChange` to toggle visibility.

### 7. Dispose

- Cancel the `requestAnimationFrame` loop
- Unsubscribe from all producer events
- Remove the `#hud` element from the DOM
- Idempotent

## Public API Surface (LOCKED contract)

```ts
export interface HUDConfig {
  rootElementId: string;   // 'hud' default
  popupDurationSec: number;  // 1.0 default
  hitFlashDurationSec: number; // 0.25 default
}

export interface HUD {
  show(): void;
  hide(): void;
  dispose(): void;
}

export function createHUD(
  player: Player,
  runLifecycle: RunLifecycle,
  planetCheckpoint: PlanetCheckpoint,
  superSuit: SuperSuitCombat,
  pickups: PickupSystem,
  config: HUDConfig,
): HUD;
```

## Tuning Knobs

| Knob | Default | Range | Category | Rationale |
|---|---|---|---|---|
| `popupDurationSec` | 1.0 | 0.5–2.0 | feel | Long enough to read, short enough to not clutter |

Tuning lives in `assets/data/hud.json`.

## Affected Systems

| System | Impact | Action Required |
|---|---|---|
| Player System | Polls `getHp`, subscribes to `onDamage` | No change |
| Run Lifecycle | Polls `getStats`, subscribes to `onStateChange` | No change |
| Planet/Checkpoint | Subscribes to `onPlanetChanged` | No change |
| Super-Suit Combat | Polls `getCooldownFraction`, subscribes to `onObstacleBroken` | No change |
| Pickup System | Subscribes to `onPickup` | No change |

## Acceptance Criteria

- [ ] On creation, HUD div is injected into `<body>`
- [ ] Distance text matches `runLifecycle.getStats().distance` each frame
- [ ] Score text updates live
- [ ] Pickup event spawns a `+10` popup that fades up and out over 1 second
- [ ] Planet-change event updates the planet label with a slide-in animation
- [ ] Super-suit cooldown ring fills smoothly over 10 s after an attack
- [ ] HUD hidden in `title`, `dead`, `results` states; visible in `running`
- [ ] Dispose removes the HUD div and cancels the rAF loop
